**Description**

The Solidity compiler accepts numeric literals of arbitrary precision. But computing those literals can take arbitrary high time.
When the following sample contract is compiled, the compiler runs for several hours (weâ€™ve not waited until it finishes):

contract BIGNUMTEST {
function bignum() {
uint c;
c=1E10000000;
}
}

**Recommendations**

Limit the size of numeric literals.


#### <img src="https://avatars.githubusercontent.com/u/9073706?v=4" width="50">[chriseth](https://github.com/chriseth) commented at [2017-12-18 11:06](https://github.com/ethereum/solidity/issues/3327#issuecomment-352395303):

Proposal: Limit the number of rational numbers to some sane value (4096 bits) and throw a FatalError otherwise.

#### <img src="https://avatars.githubusercontent.com/u/9073706?v=4" width="50">[chriseth](https://github.com/chriseth) commented at [2017-12-18 11:08](https://github.com/ethereum/solidity/issues/3327#issuecomment-352395785):

The relevant point is `RationalNumberType::binaryOperatorResult`. We cannot throw an exception there, but we can return a null pointer.
